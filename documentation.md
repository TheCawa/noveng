# Документация команд NovEng v0.4

В этом файле перечислены все команды, доступные в движке **NovEng**. Команды пишутся в файлах сценария внутри фигурных скобок: `{команда:аргумент1|аргумент2}`.

---

### 1. Визуальные эффекты и системные команды

| Команда | Аргументы | Описание |
| :--- | :--- | :--- |
| `clear` | - | Полная очистка экрана консоли и истории диалогов. |
| `shake` | `frames\|intensity` | Тряска экрана. По умолчанию: `15` кадров, сила `3`. Трясет весь текст на экране. |
| `color` | `Имя\|ANSI_Code` | Задает цвет для имени персонажа (напр. `31` — красный, `32` — зеленый, `36` — циан). |
| `next_chapter` | `file.txt` | Мгновенный переход к другому файлу сценария из `res/scenario/`. |

### 2. Звук и голос

| Команда | Аргументы | Описание |
| :--- | :--- | :--- |
| `play` | `file.mp3\|loop` | Запуск музыки из `res/music/`. Флаг `loop` зацикливает трек. |
| `stop_music` | - | Полная остановка всей фоновой музыки. |
| `sfx` | `file.wav\|pitch` | Звуковой эффект из `res/sfx/`. Можно передать высоту звука (pitch). |
| `volume` | `0.0 - 1.0` | Установка общей громкости движка (напр. `0.5` — это 50%). |
| `type` | `Имя\|pitch` | Устанавливает высоту звука "печати" (pitch) для конкретного персонажа. |

### 3. Логика, переменные и выборы

| Команда | Аргументы | Описание |
| :--- | :--- | :--- |
| `set` | `key\|value` | Создает или изменяет переменную (целое число). Вывод в тексте через `$key$`. |
| `if` | `key\|val\|f1\|f2` | Если `variables[key] >= val`, загружает `f1.txt`, иначе `f2.txt`. |
| `chance` | `%\|f1\|f2` | С вероятностью `%` (0-100) переходит к `f1`, иначе к `f2`. |
| `choice` | `T1\|F1\|T2\|F2...` | Выводит список выборов. Игрок выбирает нужный вариант нажатием цифры на клавиатуре. |

---

### Примеры использования в сценарии:

**Настройка атмосферы и персонажей:**
```
{color:Система|36}
{color:Голос|33}
{type:Система|1.2}
{type:Голос|0.7}
{play:ambient_base.mp3|loop}

[Система]
Инициализация протокола... $version$%
{sfx:beep.wav}
```

**Сюжетная развилка и проверка условий:**

```
{set:energy|10}
[Голос]
У нас хватит заряда для прыжка?

{choice:Рискнуть|jump_success.txt|Искать другой путь|search_path.txt}

// В другом файле:
{if:energy|50|super_jump.txt|fail_jump.txt}
```

**Экшен-сцена (Взрыв):**
```
[Голос]
Ложись!
{sfx:explosion.wav|0.5}
{shake:40|10}
{clear}
[Система]
Связь потеряна...
```

**Технические детали:**

**Макросы:** Любая переменная, созданная через {set:name|10}, может быть отображена в тексте как $name\$

**Выборы:** Команда choice блокирует выполнение сценария до тех пор, пока не будет нажата клавиша с цифрой от 1 до N (количество вариантов)

**Тряска:** Эффект shake работает через временное смещение offsetX и offsetY во время перерисовки всей истории сообщений.

**Динамичность:** Архитектура команд построена на паттерне Factory. Это позволяет добавлять новый функционал (новые .cpp файлы команд), просто регистрируя их через макрос REGISTER_COMMAND. Новые команды интегрируются в движок автоматически при компиляции, не требуя внесения правок в основной код.

**Распространение:** Исполняемый файл жестко привязан к структуре ресурсов. Корневая директория активов (по умолчанию res/) должна поставляться совместно с приложением. Точный путь к ресурсам определяется макросом DIR_RES в конфигурации сборки.
